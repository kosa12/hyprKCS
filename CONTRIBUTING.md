# Contributing to hyprKCS

First off, thank you for considering contributing to hyprKCS! It's people like you that make the open-source community such a great place to learn, inspire, and create.

## How Can I Contribute?

### Reporting Bugs
- **Check existing issues:** Before opening a new one, please see if the bug has already been reported.
- **Use the template:** Fill out the bug report template with as much detail as possible.
- **Include your environment:** Mention your Hyprland version, OS, and any relevant config snippets.

### Suggesting Enhancements
- **Open an issue:** Describe the feature you'd like to see and why it would be useful.
- **Be specific:** If you have ideas on how to implement it, feel free to share!

### Pull Requests
1. **Fork the repo** and create your branch from `main`.
2. **Follow existing style:** Match the project's coding conventions and formatting (run `cargo fmt`).
3. **Test your changes:** Ensure `cargo check` and `cargo build` pass without errors.
4. **Update documentation:** If you added a new feature or CLI flag, update the README.
5. **Open the PR:** Provide a clear description of what you changed and why.

## Development Setup

1. Install dependencies: `rust`, `cargo`, `gtk4`, `libadwaita`, `gtk4-layer-shell`.
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/hyprKCS.git`.
3. Build and run: `cargo run`.
